### ❌ / ⚠️ 規約違反・指摘事項

**1. Python 型ヒントの不足（複数箇所）**

```
# auth.py — login/logout に戻り値型がない
def login(body: LoginRequest, request: Request, response: Response):  # -> dict: なし
def logout(response: Response):  # -> dict: なし

# skillsheet.py
def save_skillsheet(body: SaveRequest, request: Request):  # -> dict: なし
def get_pdf(engineer_id: str, request: Request):  # -> Response: なし

# hearing.py
def _mock_chat(engineer_id: str, theme: str, messages: list, conversation_id: str):  # -> dict: なし
def _save_messages(...):  # -> None: なし

# 引数も Generic 型ヒントが未指定
messages: list  # → list[dict] が望ましい
raw_map: dict   # → dict[str, Any] が望ましい
```

**2. セキュリティ：Cookie の SameSite が strict でなく lax（重大）**

```
# routers/auth.py L46
response.set_cookie(
    key="session",
    value=token,
    httponly=True,
    max_age=8 * 60 * 60,
    samesite="lax",   # ← 規約では "strict" が必須
)
```

修正： samesite="strict" に変更してください。

**3. セキュリティ：/api/hearing/optimize にロールチェックがない**

```
# hearing.py
@router.post("/optimize")
async def optimize(body: OptimizeRequest, request: Request):
    engineer_id = body.engineer_id   # ← 任意の engineer_id を無条件で受け入れる
    # ロールチェックなし → 他エンジニアのデータを最適化できてしまう
```

修正：以下を冒頭に追加してください。

```
user = request.state.user
if user["role"] == "engineer" and user["user_id"] != engineer_id:
    raise HTTPException(status_code=403, detail="自分のデータのみ操作できます")
```

**4. セキュリティ：.gitignore が存在しない**

talentos.db と .env がリポジトリに含まれる状態です。プロジェクトルートに .gitignore を作成してください。

```
talentos.db
.env
__pycache__/
*.pyc
```

**5. SECRET_KEY がデフォルト値のまま運用される恐れ**

```
# main.py
app.state.secret_key = os.getenv("SECRET_KEY", "default-secret-key")
#                                                ↑ フォールバックが危険
```

推奨：環境変数が未設定の場合は起動を中断する。

```
secret_key = os.getenv("SECRET_KEY")
if not secret_key:
    raise RuntimeError("SECRET_KEY が設定されていません")
app.state.secret_key = secret_key
```
