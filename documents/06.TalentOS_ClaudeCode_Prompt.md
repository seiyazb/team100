## DB設計（SQLite）

以下のテーブルを `db/models.py` に定義し、`database.py` の `init_db()` 関数で CREATE TABLE IF NOT EXISTS を実行すること。

```sql
-- ユーザー管理
CREATE TABLE IF NOT EXISTS users (
    user_id       TEXT PRIMARY KEY,
    password_hash TEXT NOT NULL,
    name          TEXT NOT NULL,
    role          TEXT NOT NULL CHECK(role IN ('engineer','sales','admin'))
);

-- エンジニア基本情報（usersのengineerロール用）
CREATE TABLE IF NOT EXISTS engineers (
    engineer_id        TEXT PRIMARY KEY REFERENCES users(user_id),
    specialty          TEXT,       -- 専門分野
    relocation_ok      INTEGER DEFAULT 0,  -- 転勤可否 0/1
    work_location      TEXT,       -- 可能勤務地
    nearest_station    TEXT,       -- 最寄駅
    education_level    TEXT,       -- 最終学歴
    school_name        TEXT,       -- 学校名
    faculty_name       TEXT,       -- 学部・専攻名
    department_name    TEXT,       -- 学科名
    self_pr            TEXT,       -- 自己PR（80文字以上）
    hobbies            TEXT,       -- 趣味特技
    skill_level        TEXT,       -- スキルレベル（IT系のみ）
    language_skills    TEXT,       -- 語学力（JSON文字列）
    tool_info          TEXT,       -- ツール情報（JSON文字列）
    certifications     TEXT,       -- 資格情報（JSON文字列）
    updated_at         TEXT
);

-- スキルシート（テーマ別ヒアリング結果）
CREATE TABLE IF NOT EXISTS skill_sheets (
    sheet_id       INTEGER PRIMARY KEY AUTOINCREMENT,
    engineer_id    TEXT NOT NULL REFERENCES users(user_id),
    theme          TEXT NOT NULL CHECK(theme IN ('basic','career','skills')),
    raw_data       TEXT,   -- JSON: ヒアリング生データ
    optimized_data TEXT,   -- JSON: AI最適化後データ
    created_at     TEXT NOT NULL,
    updated_at     TEXT NOT NULL
);

-- 職務経歴（careerテーマの詳細）
CREATE TABLE IF NOT EXISTS experiences (
    exp_id         INTEGER PRIMARY KEY AUTOINCREMENT,
    engineer_id    TEXT NOT NULL REFERENCES users(user_id),
    project_name   TEXT,
    period_start   TEXT,   -- YYYY/MM
    period_end     TEXT,   -- YYYY/MM
    team_size      INTEGER,
    role_title     TEXT,
    tech_stack     TEXT,   -- JSON配列文字列
    description    TEXT,
    created_at     TEXT
);

-- ヒアリングログ（会話履歴）
CREATE TABLE IF NOT EXISTS hearing_logs (
    log_id         INTEGER PRIMARY KEY AUTOINCREMENT,
    engineer_id    TEXT NOT NULL REFERENCES users(user_id),
    theme          TEXT NOT NULL,
    messages       TEXT NOT NULL,  -- JSON: [{role, content, timestamp}, ...]
    completed      INTEGER DEFAULT 0,
    completed_at   TEXT,
    dify_conversation_id TEXT
);
```

## 認証仕様

### ログイン

**エンドポイント**：`POST /api/auth/login`

リクエスト：
```json
{ "user_id": "engineer01", "password": "pass123" }
```

処理：
1. DBで user_id を検索
2. `passlib` で password_hash と照合
3. 一致したら `HttpOnly` Cookie に session トークンを発行（有効期限8時間）
4. トークンには `{"user_id": "...", "name": "...", "role": "..."}` を格納

レスポンス：
```json
{ "success": true, "role": "engineer", "name": "山田 太郎" }
```

失敗時：
```json
{ "success": false, "message": "IDまたはパスワードが正しくありません" }
```

### ログアウト

**エンドポイント**：`POST /api/auth/logout`
- Cookie を削除してリダイレクト（`/login` へ）

### セッション確認ミドルウェア

`main.py` に以下のミドルウェアを追加：
- `/login` と `/static/**` 以外のルートはすべて Cookie セッションを検証
- 未認証の場合は `302 → /login` にリダイレクト
- 認証済みの場合はリクエストに `request.state.user`（user_id, name, role）をセット

## 概要

エンジニアがAIとチャットしながらスキルシートを作成する画面。
テーマ（基本情報 → 経歴 → スキル）に分けてヒアリングし、各テーマ完了時に右ペインのスキルシートプレビューを更新する。

## エンドポイント

### `POST /api/hearing/chat`

リクエスト：
```json
{
  "theme": "career",
  "message": "テックリードとして8名のチームを...",
  "conversation_id": "abc123"
}
```

処理：
1. `hearing_logs` に会話を保存
2. Dify チャットAPI を呼ぶ（後述のDify連携仕様を参照）
3. Dify が返した `answer` を解析：
   - `theme_completed: true` が含まれる場合 → `skill_sheets` テーブルに `extracted_data` を保存
4. レスポンスを返す

レスポンス（テーマ未完了時）：
```json
{
  "message": "ありがとうございます。期間はどのくらいでしたか？",
  "theme_completed": false,
  "sheet_update": null,
  "conversation_id": "abc123"
}
```

レスポンス（テーマ完了時）：
```json
{
  "message": "経歴のヒアリングが完了しました。",
  "theme_completed": true,
  "conversation_id": "abc123",
  "sheet_update": {
    "theme": "career",
    "data": {
      "project_name": "金融システムAWS移行",
      "role_title": "テックリード",
      "team_size": 8,
      "period_start": "2023/04",
      "period_end": "2024/09",
      "tech_stack": ["AWS", "Terraform", "GitHub Actions"],
      "description": "オンプレミスからAWSへの移行をリード"
    }
  }
}
```

### `POST /api/hearing/optimize`

リクエスト：
```json
{ "engineer_id": "engineer01" }
```

処理：
1. `skill_sheets` から当該エンジニアの全テーマデータを取得
2. Dify 最適化ワークフローAPIを呼ぶ（後述）
3. 最適化結果を `skill_sheets.optimized_data` に保存

レスポンス：
```json
{
  "success": true,
  "optimized": {
    "career": { "description": "...", "suggestions": ["...", "..."] },
    "skills": { ... }
  }
}
```

## Dify連携仕様（フェーズ2時点）

`.env` の `DIFY_API_KEY` と `DIFY_BASE_URL` が空の場合はモック応答を返すこと。

**モック応答の仕様**：
- 会話が5ターン未満 → `theme_completed: false`、適当な追加質問を返す
- 会話が5ターン以上 → `theme_completed: true`、ダミーの `extracted_data` を返す
- 最適化API → そのまま入力データを `optimized_data` として返す（変換なし）

**本番Dify連携**（`.env` に値がある場合）：

チャットAPI呼び出し：
```python
POST {DIFY_BASE_URL}/v1/chat-messages
Headers: Authorization: Bearer {DIFY_API_KEY}
Body: {
  "query": "{ユーザーメッセージ}",
  "conversation_id": "{conversation_id}",  # 空文字列 = 新規会話
  "inputs": {
    "theme": "{theme}",
    "required_fields": "{テーマ別の必須項目リスト}"
  },
  "response_mode": "blocking",
  "user": "{engineer_id}"
}
```

テーマ別の required_fields：
- `basic`：専門分野、転勤可否、可能勤務地、最寄駅、最終学歴、学校名、学部・専攻名、学科名、自己PR、趣味特技、スキルレベル
- `career`：プロジェクト名、期間（開始・終了）、チーム規模、役職・役割、使用技術、業務内容
- `skills`：使用ツール、保有資格、語学力

最適化ワークフローAPI呼び出し：
```python
POST {DIFY_BASE_URL}/v1/workflows/run
Headers: Authorization: Bearer {DIFY_API_KEY}
Body: {
  "inputs": {
    "career_data": "{career_raw_data_json}",
    "skills_data": "{skills_raw_data_json}",
    "basic_data": "{basic_raw_data_json}"
  },
  "response_mode": "blocking",
  "user": "{engineer_id}"
}
```

## JS仕様（hearing.js）

```javascript
// グローバル状態
const state = {
  conversationId: "",
  currentTheme: "basic",     // basic → career → skills の順
  themeStatus: { basic: "pending", career: "pending", skills: "pending" },
  isLoading: false
};

// テーマの順序
const THEME_ORDER = ["basic", "career", "skills"];
const THEME_LABELS = { basic: "基本情報", career: "経歴", skills: "スキル・資格" };

// 初回メッセージ（DOMロード時に自動表示）
const INITIAL_MESSAGES = {
  basic:  "こんにちは。スキルシートを一緒に作成しましょう。まず、あなたのお名前と専門分野、転勤の可否について教えてください。",
  career: "基本情報の入力が完了しました。次は職務経歴についてお聞きします。直近のプロジェクトから教えてください。",
  skills: "経歴の入力が完了しました。最後に、保有スキル・資格・語学力についてお聞きします。"
};

// 送信処理
async function sendMessage(text) {
  // 1. ユーザーメッセージをUIに追加
  // 2. タイピングインジケーター表示
  // 3. POST /api/hearing/chat
  // 4. タイピングインジケーター非表示
  // 5. AIメッセージをUIに追加
  // 6. theme_completed === true の場合：
  //    a. 右サイドバーのスキル評価を更新（sheet_update.data を元に）
```

### エンドポイント：`POST /api/search`

リクエスト：
```json

```

処理：
1. Dify 検索クエリ解析ワークフロー を呼ぶ

Dify呼び出し：
```python
POST {DIFY_BASE_URL}/v1/workflows/run
Body: {
  "inputs": { "search_query": "{query}" },
  "response_mode": "blocking",
  "user": "{current_user_id}"
}
```

Difyが返す想定フォーマット：
```json
{
  "outputs": {
    "conditions": {
      "skills": ["Python", "AWS"],
      "min_experience_years": 3,
      "remote": true
    },
    "ai_insight": "Python・AWS経験者でリモート可能なエンジニアを検索しました。"
  }
}
```

2. `conditions` を元に SQLite を検索：
   - `skills` → `skill_sheets.raw_data` の tech_stack に含まれる
   - `remote` → `engineers.relocation_ok = 1`
3. マッチしたエンジニア一覧を返す

**モック動作**（Dify未設定時）：
- クエリに含まれるキーワードで tech_stack を LIKE 検索する
